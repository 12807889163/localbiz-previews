🌐 Custom Domain Setup Guide

Namecheap DNS → GitHub Pages — ~10 minutes total

This guide shows you how to connect your own domain (e.g. yourbusiness.com) to your new website hosted on GitHub Pages. We recommend Namecheap — but the DNS steps are similar on GoDaddy, Google Domains, or Cloudflare.

⏱️ Total time: ~10 minutes  |  💰 Cost: ~$12/year

Jump to step:

1. Buy Domain 2. Namecheap DNS 3. GitHub Pages 4. Enable HTTPS

Step 1 — Buy Your Domain

🌐
Buy a domain (Namecheap recommended)

Go to namecheap.com and search for your desired domain. Most domains cost $10–15/year for the first year.

Good options to check:

• yourbusiness.com — the classic choice
• yourbusiness.services — great for service businesses
• yourbusiness.xyz — budget-friendly

Complete your purchase and move on to Step 2.

Step 2 — Set Up DNS on Namecheap

🔧
Add DNS records in Namecheap

After purchasing your domain, log into Namecheap and go to Dashboard → Domain List → Manage → Advanced DNS.

📸 Screenshot: Namecheap Advanced DNS page

Look for the "HOST RECORDS" section near the bottom of the page

Delete any existing A records (some registrars add defaults). Then add the following records:

CNAME Record (required — points to your GitHub site)

TypeHostValueTTL
CNAMEwww12807889163.github.ioAutomatic

ALIAS Record (optional but recommended — root domain redirect)

TypeHostValueTTL
ALIAS@12807889163.github.ioAutomatic
⚠️ Important: Use ALIAS, not CNAME Some registrars call this "ALIAS", "URL Redirect", or "Forwarding". It must point to 12807889163.github.io — not your GitHub username. If you don't see ALIAS, use URL Redirect/Forwarding to forward your root domain to www.yourdomain.com.
✅ DNS tip: Changes can take 10 minutes to 48 hours to propagate globally. Don't panic if things don't work immediately — give it up to 2 hours before troubleshooting.

Step 3 — Enable GitHub Pages + Custom Domain

⚙️
Tell GitHub to use your custom domain

You'll need access to the GitHub repository. If you don't have it, reply to your delivery email and we'll add you as a collaborator.

Steps:

1. Go to the GitHub repository: 12807889163/localbiz-previews
2. Go to Settings → Pages
3. Under "Custom domain", enter your domain (e.g. yourbusiness.com)
4. Click Save

📸 Screenshot: GitHub Pages Settings

Settings → Pages → Custom domain field highlighted

⚠️ DNS check: GitHub will check your DNS configuration. If it shows a warning, the DNS may not have propagated yet — wait 10–30 minutes and check again.

Step 4 — Enable HTTPS (Recommended)

🔒
Secure your site with a free SSL certificate

GitHub Pages automatically provisions a free SSL certificate via Let's Encrypt for custom domains.

In your GitHub repository Settings → Pages:

✅ Check "Enforce HTTPS"
This may take 5–15 minutes after initial setup.

✅ That's it! Once HTTPS is enforced, your site will be live at https://yourbusiness.com — no "not secure" warnings, great for Google rankings.
