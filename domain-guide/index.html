🌐 How to Connect Your Domain

Step-by-step guide for beginners — no technical experience needed

This guide walks you through connecting your own domain name (like yourbusiness.com) to your new website. We'll do this together, one click at a time. Don't worry — you don't need to know anything technical to follow along.

⏱️ Total time: about 10–15 minutes  |  💰 Annual cost: ~$12/year

Before You Start

Here's what you'll need:

  • Your domain name (e.g. yourbusiness.com) — if you haven't bought one yet, we'll cover that in Step 1
  • Login to your Namecheap account — or we'll create one together in Step 1
  • Access to your email inbox (we'll send you a link to get GitHub access)
  • About 10–15 minutes without interruptions

Step 3 — Set Up Your DNS Records

💡 What is DNS? Let me explain in plain English:

DNS is like the phone book of the internet. When someone types yourbusiness.com into their browser, DNS tells the internet where to find your website. We're going to set up two "listings" in that phone book so visitors can find your site.

1

Go to Advanced DNS settings

While on the domain management page, look for a tab called Advanced DNS. Click it.

⚙️
What you should see
A page with several sections including "HOST RECORDS", "EMAIL RECORDS", and "URL FORWARDING". We're working in the HOST RECORDS section.
2

Delete any existing A records (if present)

Look in the HOST RECORDS section. Sometimes Namecheap adds default A records for you — we need to remove those first.

Next to each existing A record, you'll see a Delete button (trash icon 🗑️). Click it for every A record you see.

⚠️ Don't delete CNAME records — only delete A records for now.

If you're not sure which are A records, look at the "Type" column — it will say "A" for A records. Leave everything else alone.

3

Add a CNAME record (for www.yourdomain.com)

A CNAME record tells the internet: "When someone visits www.yourbusiness.com, send them to my website."

Look for a button that says Add New Record. Click it, then fill in these three fields:

Type — click the dropdown and select CNAME

Host — type www

Value — type 12807889163.github.io

TTL — leave as Automatic

Then click Save All Changes.

📝
What the filled-in record should look like
Type: CNAME | Host: www | Value: 12807889163.github.io | TTL: Automatic
4

Add an ALIAS record (for yourdomain.com — the root)

This step makes sure visitors who type yourbusiness.com (without the "www") also end up at your website.

Click Add New Record again, then:

Type — click the dropdown and select ALIAS

Host — type the symbol @ (just the one character — no quotes)

Value — type 12807889163.github.io

TTL — leave as Automatic

Then click Save All Changes.

📝
What the filled-in ALIAS record should look like
Type: ALIAS | Host: @ | Value: 12807889163.github.io | TTL: Automatic
⚠️ Don't see "ALIAS" in the Type dropdown?

Some Namecheap accounts don't have ALIAS as an option. If that's the case, scroll down to the URL FORWARDING section instead. Set it up like this:

Source: @ (just the @ symbol) → Destination: http://www.yourbusiness.com → 301 (Permanent) redirect

✅ DNS records are set!

Your changes are saved. It can take anywhere from 10 minutes to 48 hours for these changes to spread across the internet. Most of the time it's fast — but don't worry if it takes a couple hours.

Step 5 — Connect Your Domain in GitHub

1

Go to the repository settings

After accepting the GitHub invitation, you should be on the website repository page. It will look something like github.com/12807889163/localbiz-previews.

At the top of the page, click the Settings tab.

⚙️
What you should see
The repository page. Settings is at the top in the grey navigation bar, next to "Actions", "Projects", etc.
2

Find the Pages section

In the left-hand sidebar of Settings, scroll down until you see a section called Pages. Click on it.

📄
What you should see
The Pages settings page. It will say "GitHub Pages" at the top with a subtitle about enabling pages.
3

Enter your custom domain

Look for a field called Custom domain (it will be a text input box).

Type your domain name there — for example: yourbusiness.com

Then click Save.

🌐
What you should see
The Custom domain field filled in with something like "yourbusiness.com". There's a blue Save button to the right.

GitHub will automatically check if your DNS is set up correctly. This check might show a yellow warning at first — that's normal. It means DNS hasn't fully propagated yet. Give it 10–30 minutes and check again.

Step 6 — Enable HTTPS (Secure the Site)

💡 What is HTTPS? In plain English:

HTTPS is what puts the little padlock 🔒 in your browser's address bar. It means the connection between your visitor's browser and your website is private and secure. Without it, browsers show a "Not Secure" warning — which you definitely don't want!

1

Check the "Enforce HTTPS" box

You're still on the GitHub Pages settings page from Step 5. Below the custom domain field, look for a checkbox that says "Enforce HTTPS".

Click the checkbox so it's ticked ✅.

🔒
What you should see
A checkbox labeled "Enforce HTTPS" — it will be below or next to the Custom domain field. Make sure it's checked/ticked.

GitHub will automatically get a free SSL certificate (that's the padlock 🔒) for your domain. This can take anywhere from 5 minutes to 24 hours.

✅ You're all done!

Once HTTPS is active, your website will be live at https://yourbusiness.com. No more "not secure" warnings — just a clean, professional site your customers will trust.

Troubleshooting — "It Doesn't Work Yet"

Here's what to check if you've finished all the steps but your site isn't showing up yet:

🔴 "GitHub says DNS check failed"

Wait 10–30 minutes and try again. DNS changes can take time to propagate. If it still fails after an hour, double-check your DNS records match exactly what we wrote in Step 3 — especially the spelling of 12807889163.github.io.

🔴 "I see a GitHub 404 error"

This usually means the domain was saved in GitHub before DNS fully propagated. Go back to Settings → Pages, delete the custom domain, click Save, wait 2 minutes, then re-enter it and save again.

🔴 "My site loads but shows 'Not Secure'"

Go back to Settings → Pages and make sure the "Enforce HTTPS" checkbox is ticked. If it is, wait — it can take up to 24 hours for the SSL certificate to activate after you first enable it.

🔴 "www.yourbusiness.com works but yourbusiness.com doesn't"

The ALIAS record for @ didn't take. Go back to Namecheap → Advanced DNS and check that the ALIAS record (or URL forwarding rule) is still there. If it's gone, re-add it following Step 3, Step 4.

🔴 "I've waited 48 hours and nothing works"

Reply to your delivery confirmation email and let us know. We'll help you figure out what's wrong — at no extra charge. That's what we do.
